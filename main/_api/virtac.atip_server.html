ATIPServer

A soft-ioc server allowing ATIP to be interfaced using EPICS, in the same manner as the live machine.
class virtac.atip_server.ATIPServer(ring_mode, limits_csv=None, bba_csv=None, feedback_csv=None, mirror_csv=None, tune_csv=None, disable_emittance=False)[source]#

A soft-ioc server allowing ATIP to be interfaced using EPICS, in the same manner as the live machine.

Attributes

lattice#

An instance of a Pytac lattice with a simulator data source.

Type:

pytac.lattice.Lattice

tune_feedback_status#

A boolean indicating whether the tune feedback records have been created and the monitoring systems are running.

Type:

bool

Parameters:

  • ring_mode (str) – The ring mode to create the lattice in.

  • limits_csv (str) – The filepath to the .csv file from which to load the pv limits, for more information see create_csv.py.

  • bba_csv (str) – The filepath to the .csv file from which to load the bba records, for more information see create_csv.py.

  • feedback_csv (str) – The filepath to the .csv file from which to load the feedback records, for more information see create_csv.py.

  • mirror_csv (str) – The filepath to the .csv file from which to load the mirror records, for more information see create_csv.py.

  • tune_csv (str) – The filepath to the .csv file from which to load the tune feedback records, for more information see create_csv.py.

  • disable_emittance (bool) – Whether the emittance should be disabled.

update_pvs()[source]#

The callback function passed to ATSimulator during lattice creation, it is called each time a calculation of physics data is completed. It updates all the in records that do not have a corresponding out record with the latest values from the simulator.

monitor_mirrored_pvs()[source]#

Start monitoring the input PVs for mirrored records, so that they can update their value on change.

refresh_record(pv_name)[source]#

For a given PV refresh the time-stamp of the associated record, this is done by setting the record to its current value.

Parameters:

pv_name (str) – The name of the record to refresh.

setup_tune_feedback(tune_csv=None)[source]#

Read the tune feedback .csv and find the associated offset PVs, before starting monitoring them for a change to mimic the behaviour of the quadrupoles used by the tune feedback system on the live machine.

Note

This is intended to be on the recieving end of the tune feedback system and doesn’t actually perfom tune feedback itself.

Parameters:

tune_csv (str) – A path to a tune feedback .csv file to be used instead of the default filepath passed at startup.

stop_all_monitoring()[source]#

Stop monitoring mirrored records and tune feedback offsets.

set_feedback_record(index, field, value)[source]#

Set a value to the feedback in records.

possible element fields are:
[‘error_sum’, ‘enabled’, ‘state’, ‘offset’, ‘golden_offset’, ‘bcd_offset’,

‘bba_offset’]

possible lattice fields are:
[‘beam_current’, ‘feedback_status’, ‘bpm_id’, ‘emittance_status’,

‘fofb_status’, ‘cell_<cell_number>_excite_start_times’, ‘cell_<cell_number>_excite_amps’, ‘cell_<cell_number>_excite_deltas’, ‘cell_<cell_number>_excite_ticks’, ‘cell_<cell_number>_excite_prime’]

Parameters:

  • index (int) – The index of the element on which to set the value; starting from 1, 0 is used to set on the lattice.

  • field (str) – The field to set the value to.

  • value (number) – The value to be set.

Raises:

pytac.FieldException – If the lattice or element does not have the specified field.
